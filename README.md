# Chronosphere MCMC: Космологический анализ модели Хроносферы

Данный пакет реализует байесовский анализ (MCMC) модели Хроносферы с использованием современных космологических данных (Planck 2018, Pantheon+, DESI BAO). Код построен на основе библиотек `cobaya` и `CLASS`.

## 🔧 Требования

- Python 3.8 или выше
- Библиотеки:
  - `classy` (интерфейс к CLASS)
  - `cobaya` (байесовский анализ)
  - `getdist` (анализ MCMC цепочек)
  - `numpy`, `scipy`, `matplotlib`

## 📦 Установка

1. **Клонируйте репозиторий** (или скопируйте файл `chronosphere_mcmc.py`):
```bash
git clone https://github.com/chronosphere/cosmo-mcmc.git
cd cosmo-mcmc
Создайте виртуальное окружение (рекомендуется):

## 📊 Параметры модели
Параметр  Описание	                         Приор	       По умолчанию
h	        Постоянная Хаббла	                 [0.5, 0.9]	   0.67
omega_b	  Плотность барионов	               [0.005, 0.1]	 0.022
omega_cdm	Плотность тёмной материи	         [0.05, 0.4]	 0.12
tau	      Оптическая толщина реионизации	   [0.01, 0.1]	 0.054
A_s	      Амплитуда первичных возмущений	   [1e-9, 5e-9]	 2.1e-9
n_s	      Спектральный индекс	               [0.8, 1.2]	   0.965
epsilon0	Современное значение ε = a/R	     [0.1, 0.5]	   0.32
alpha	    Параметр потенциала (связь с w0)	 [0.0, 1.0]	   0.5
beta	    Параметр потенциала (связь с wa)	 [-0.5, 0.5]	 0.0

Производные параметры:

w0 = -1 + alpha * epsilon0² — уравнение состояния при z=0

wa = beta * epsilon0 — параметр наклона уравнения состояния

##🔌 Настройка данных
В текущей версии likelihood'ы используют заглушки (lambda data, **kwargs: None). Чтобы использовать реальные данные:

Замените имена likelihood'ов на корректные имена, известные cobaya:

planck_2018 → planck_2018_highl_plik или planck_2018_lowl_tt

pantheon_plus → sn.pantheonplus (требуется пакет cobaya-sn)

desi_bao → bao.boss_desi

Убедитесь, что необходимые данные установлены (cobaya загрузит их автоматически при первом запуске).

При необходимости добавьте дополнительные likelihood'ы:

python
"likelihood": {
    "planck_2018_highl_plik": None,
    "planck_2018_lowl_tt": None,
    "planck_2018_lowl_ee": None,
    "planck_2018_lensing": None,
    "sn.pantheonplus": None,
    "bao.boss_desi": None,
}

## 🧩 Расширение и модификация
Добавление нового параметра
Добавьте параметр в словарь params в info

Если параметр влияет на CLASS, передайте его в class_params внутри calculate

Если параметр является производным, определите его через derived с лямбда-функцией

Модификация потенциала
Для точной реализации потенциала дилатона (из раздела 3 статьи) замените связь w0 и wa с epsilon0 на более точную, основанную на численном решении уравнения движения.

Использование с суперкомпьютером
Для больших MCMC (50000+ шагов) рекомендуется:

Увеличить max_samples до 100000-200000

Установить Rminus1_stop: 0.02 для более быстрой сходимости

Использовать MPI для параллельного запуска (cobaya поддерживает mpi)
